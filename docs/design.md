# UI Pro Max Design Baseline

## 设计目标
这轮改造的目标不是把页面做得更花，而是把产品收敛成一套稳定、专业、可持续扩展的文本线索工作台。

核心原则：
- 产品目标是从文本中提取线索、结构和 insight，不把界面默认心智框成“确认是不是同一个作者”。
- 继续使用现有 OKLCH 主题变量、深浅主题和 shadcn 组件体系，不新造第二套品牌系统。
- 视觉追求专业度和可读性，不依赖渐变、噪声层、装饰性图标和过度动画。
- 页面必须按“摘要页 + 专家页”分层，避免同一功能在多个页面完整重复。

## 视觉系统
### 色彩语义
- 蓝色：主线发现、综合判断、进行中状态。
- 琥珀：证据、限制、需要留意的反向线索。
- 紫色：写作特征画像与多维度行为轮廓。
- 绿色：聚类、分组、结构稳定性。
- 红色：异常、冲突、需要复核的离群点。
- 灰蓝：叙事章节、附录、辅助信息。

### 界面约束
- 禁止渐变设计。所有图表与容器都使用纯色、离散色阶和清晰边框。
- 不使用多余图标。只有在导航、刷新、取消这类系统动作上保留必要图标。
- 同一屏幕不超过 3 层容器语义：页面壳层、主信息层、嵌套细节层。
- 阴影只用于轻微分层，不承担装饰角色。

### 动效原则
- 仅保留页面进入、区块渐入、hover 强化和打开抽屉这类解释状态的动效。
- 不使用持续性装饰动效、浮动背景颗粒或大面积动画层。
- **例外：surface-flow 背景层**。全视口多层渐变动画作为环境纹理保留，提供品牌辨识度和空间感。该系统遵循单独的约束：仅使用 `transform` + `opacity` 保持 GPU 合成；`prefers-reduced-motion` 下完全禁用；移动端大幅降低漂移幅度和不透明度。详见 `globals.css` 中 `.app-surface-flow` 相关实现。

## 信息架构
### `/analyses`
- 任务索引页。
- 负责总览、筛选、检索、进入具体任务。
- 不负责报告阅读，也不展示完整深度分析。

### `/analyses/new`
- 唯一的新建入口。
- 负责上传数据、配置策略、填写案件信息和启动任务。
- 只展示数据就绪摘要，不展示报告型可视化。

### `/analyses/detail`
- 唯一主阅读页。
- 运行态负责进度、日志、连接状态和取消任务。
- 完成态负责卷轴式综合阅读、证据链、聚类与异常入口、Agent 备注和 QA。

### `/analyses/features`
- 唯一的特征专家页。
- 负责特征分布、样本群组比较、相似度热力图和原始特征查看。
- 不承担最终阅读入口，不重复主报告页的综合判断。

### `/settings`
- 系统设置摘要页。
- 只负责默认参数、Prompt 覆盖和 backend 摘要。

### `/settings/backends`
- backend 专家工作台。
- 负责新建、编辑、删除、分组、连通性测试和密钥绑定。

## 去重规则
- 详情页是综合发现的唯一入口。
- 特征页只做专家级验证和比较，不重复展示综合结论。
- 设置首页只放 backend 摘要与入口，不嵌入完整 CRUD。
- 历史页只做任务检索和恢复，不做报告摘要阅读。

## 垂直可视化方案
本轮报告页采用 `Scroll + Linked Lenses`。

### 卷轴结构
1. Case Header
2. Conclusion Rail
3. Evidence Chain
4. Writing Profiles
5. Cluster View
6. Narrative Spine
7. Appendix

阅读路径固定为：
- 先看焦点判断。
- 再看证据链确认支撑与反向线索。
- 然后决定是否需要打开画像、聚类或附录。

### 交互模型
- `hover`：预览上下文，不打断阅读。
- `click`：打开右侧抽屉查看实体详情。
- `scroll`：保持主阅读流连续，不把用户切成过多 tabs。

### 透镜机制
- Feature lens：在证据和文本层补充关键特征签名。
- Cluster lens：把画像和文本联动到分组结构。
- Anomaly lens：把异常样本高亮叠加到证据与聚类视图。

透镜默认关闭，只在用户主动开启时叠加额外信息层。

### FocusContext
- 点击任意结论、证据、文本、画像或聚类后进入聚焦态。
- 相关元素保持高亮，无关元素降低透明度。
- 聚焦可随时清除，避免界面长期保持“被锁住”。

## 页面结构说明
### 详情页
1. 顶部工具条：返回、刷新、取消、导出、进入特征工作台、重跑。
2. 任务头：任务状态、文本数、来源组数、模型、案件元数据、请求范围。
3. 运行态：进度卡。
4. 完成态：卷轴式主阅读区。
5. 卷轴后：异常样本表、Agent 备注、QA。

### 特征页
1. 顶部工作台说明与返回入口。
2. 上下文和关键统计。
3. 左侧控制栏：来源组筛选、特征组选择、特征选择、清除联动。
4. 右侧主区：分布图、跨信号比较、相似度热力图、原始特征查看。

## 响应式策略
- 桌面优先：详情页和特征页采用“侧栏 + 主区”。
- 窄屏下降级为单列，不允许主内容横向滚动破坏阅读。
- sticky 侧栏在小屏自动退化为普通流布局。

## 验收标准
### 什么算美观实用
- 第一屏能回答“这里看什么”。
- 强调色只用于真正重要的发现、状态和交互反馈。
- 图表不靠视觉噱头，而是帮助快速判断结构和异常。
- 主阅读流连续，用户不需要先学会 UI 才能读懂内容。

### 什么算硬搞 UI
- 用渐变、光效、装饰层制造“高级感”。
- 用多个小卡片把一个意思拆碎。
- 在主阅读页和专家页重复展示同一能力。
- 让图表像作品集，而不是分析工具。

## 后续扩展规则
- 新模块上线前，先定义它属于摘要页还是专家页。
- 新可视化优先挂在专家页或卷轴内部，不回退成大量零散面板。
- 新任务文案优先使用“样本、来源组、信号、线索、结构、异常”等词，不默认使用“作者归属”框架。
- 所有新增图表继续遵守无渐变、离散色阶、低图标密度和扁平容器规则。
