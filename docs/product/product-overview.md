# 产品概述

## 产品定位

`text` 是一个面向数字取证与文本研判场景的多智能体分析平台。系统结合 Rust 高性能特征提取、Python NLP 管道、LLM 多智能体分析与 Web/Tauri 交互界面，为内部分析人员提供可复核、可扩展的文本取证工作流。

它解决的核心问题不是“生成一段通用总结”，而是围绕文本证据形成结构化研判结论，包括：

- 作者归属验证
- 候选作者识别
- 写作画像分析
- 傀儡账号或同人操控识别
- 综合取证报告生成

## 目标用户

当前文档面向内部团队，因此产品视角也按内部使用角色描述：

- 取证分析人员：上传文本、配置任务、阅读报告
- 研发人员：扩展分析能力、维护数据链路、优化时延
- 交付人员：打包桌面版本、配置后端、支持现场部署
- 技术负责人：把控系统边界、能力覆盖与质量要求

## 核心能力

### 1. 多任务文本取证

系统当前支持以下任务类型：

- `full`: 综合分析
- `verification`: 作者验证
- `closed_set_id`: 闭集候选识别
- `open_set_id`: 开集候选识别
- `clustering`: 无监督聚类
- `profiling`: 写作画像
- `sockpuppet`: 傀儡账号检测

### 2. 双层特征提取

系统先执行确定性的文本特征计算，再交由多智能体解释：

- Rust 层负责词汇、句法、n-gram、Unicode 等高吞吐特征
- Python 层负责 spaCy、嵌入、LIWC 等 NLP 特征

### 3. 多智能体协同分析

系统不是单模型单次输出，而是由多个学科视角并行生成报告，再由综合 Agent 汇总：

- Stylometry
- Writing Process / Psycholinguistics
- Sociolinguistics
- Computational
- Synthesis

### 4. 多入口使用方式

系统同时提供三种使用入口：

- CLI：适合批处理和脚本化分析
- FastAPI：适合 Web 前端或外部系统接入
- Web/Tauri：适合内部交互式使用和桌面交付

## 典型使用场景

### 作者验证

分析人员提供可疑文本与参考作者样本，系统输出支持或反对同一作者假设的结论与证据。

### 账号同人操控判断

分析人员上传账号文本、活动事件和交互关系，系统结合语言特征与行为上下文评估是否存在共同控制。

### 写作画像与群体差异分析

在缺少明确身份标签时，系统可从文本中提炼风格与认知特征，用于辅助分群、标注和研判。

### 交互式报告审阅

通过 Web 或桌面端查看分析进度、结构化指标、异常样本和综合结论，便于复核与导出。

## 输入与输出

### 输入

系统支持以下输入对象：

- 文本集合 `texts`
- 证据制品 `artifacts`
- 活动事件 `activity_events`
- 交互关系 `interaction_edges`
- 案件上下文 `case_metadata`

常见文件格式包括：

- `.txt`
- `.csv`
- `.json`
- `.jsonl`
- 目录批量导入

### 输出

系统输出包括：

- 特征向量
- 各学科 Agent 报告
- 综合结论与置信度
- 进度事件流
- Markdown / JSON / Rich 报告

## 产品边界

当前系统主要聚焦“文本取证分析平台”本身，不负责：

- 通用数据采集系统建设
- 大规模训练平台或模型标注平台
- 证据链外部存储治理
- 完整案件管理系统

因此，产品设计更偏向“分析引擎 + 交互工作台”，而不是完整办案平台。
