# 产品概述

## 产品定位

text —— 文本侦探。给它任何文本，它像侦探一样拉出隐藏线索。

系统结合 Rust 高性能特征提取、Python NLP 管道、LLM 多智能体推理与 Web/Tauri 交互界面，围绕文本形成结构化洞察。核心能力包括：

- 从文本中发现可追踪的语言信号、异常点和隐藏线索
- 提炼写作者的风格指纹、心理画像和社会身份特征
- 比较多来源文本之间的稳定差异与重合区域
- 发现潜在的聚类关系、群组边界和共控迹象
- 生成结构化、可导出的综合研判报告

产品遵循**渐进披露**原则：所有用户从同一个入口进入，差异在于选择探索的深度——浏览线索卡片、深入写作 DNA 画像、或展开完整调查。

## 目标用户

### 普通用户——文字 DNA 自我发现

对自己写作风格好奇的人。粘贴文本，获得写作指纹、心理倾向和表达习惯的画像报告，体验类似 MBTI 式的自我认知。

### OSINT 调查者——线索提取与文本情报

需要从未知来源文本中提取有价值线索的调查人员。系统提供作者可能的教育背景、职业领域、语言社群特征、可追踪语言标记和异常信号。

### 专业分析人员——归属分析与深度研判

执行作者归属、傀儡账号检测、文本聚类等专业任务的分析人员。上传文本、配置任务参数、阅读多智能体交叉验证报告。

## 核心能力

### 1. 多任务文本分析

系统支持 9 种任务类型：

- `full`: 综合分析
- `verification`: 来源比对
- `closed_set_id`: 闭集候选识别
- `open_set_id`: 开集候选识别
- `clustering`: 无监督聚类
- `profiling`: 写作画像
- `sockpuppet`: 傀儡账号检测
- `self_discovery`: 文字 DNA 自我发现
- `clue_extraction`: 线索提取

### 2. 三层发现结构

所有 Agent 的输出统一包含三个层次，任何任务类型都不会隐藏或省略：

- **clue（线索）**：可追踪的信号、异常点、模式——通用入口
- **portrait（画像）**：写作者特征、习惯解读——偏自我发现
- **evidence（证据）**：支撑数据和文本摘录——偏专业/OSINT

不同任务类型的差异仅在于**叙事视角**：`self_discovery` 使用第二人称（"你的写作显示……"），其余使用第三人称（"该文本作者表现出……"）。

### 3. 双层特征提取

系统先执行确定性的文本特征计算，再交由多智能体解释：

- Rust 层负责词汇、句法、n-gram、Unicode 等高吞吐特征
- Python 层负责 spaCy、嵌入、LIWC 等 NLP 特征

### 4. 多智能体协同分析

多个学科视角并行生成报告，再由综合 Agent 汇总：

- **Stylometry**：写作指纹——标志性表达、一致性模式、风格习惯
- **Psycholinguistics**：心理画像——思维风格、情感倾向、认知模式
- **Sociolinguistics**：社会身份线索——教育背景、职业领域、文化社群特征
- **Computational**：模式与关联——隐藏数据模式、异常点、跨文本关联信号
- **Synthesis**：侦探总结——交叉验证所有线索、矛盾识别、置信度校准

### 5. 多入口使用方式

系统同时提供三种使用入口：

- CLI：适合批处理和脚本化分析
- FastAPI：适合 Web 前端或外部系统接入
- Web/Tauri：适合交互式使用和桌面交付

## 典型使用场景

### 文字 DNA 自我发现

用户粘贴自己的文本，系统生成写作指纹画像：风格习惯、心理倾向、表达模式和社会身份线索。以第二人称呈现，友好可读。

### OSINT 线索提取

调查者提供未知来源文本，系统提取作者可能的背景特征、可追踪的语言标记和异常信号，以第三人称客观分析呈现，类似情报简报。

### 来源比对

分析人员提供目标文本与参考来源组样本，系统输出支持或反对某种解释路径的证据，而不是只返回单一身份判断。

### 账号共控判断

分析人员上传账号文本、活动事件和交互关系，系统结合语言特征与行为上下文评估是否存在共同控制。

### 写作画像与群体差异分析

在缺少明确身份标签时，系统可从文本中提炼风格与认知特征，用于辅助分群、标注和研判。

### 交互式报告审阅

通过 Web 或桌面端查看分析进度、结构化指标、异常样本和综合结论，便于复核与导出。

## 输入与输出

### 输入

系统支持以下输入对象：

- 文本集合 `texts`
- 证据制品 `artifacts`
- 活动事件 `activity_events`
- 交互关系 `interaction_edges`
- 案件上下文 `case_metadata`

常见文件格式包括：

- `.txt`
- `.csv`
- `.json`
- `.jsonl`
- 目录批量导入

### 输出

系统输出包括：

- 特征向量
- 各学科 Agent 报告（含 clue / portrait / evidence 三层结构）
- 综合结论与置信度
- 进度事件流
- Markdown / JSON / Rich 报告

## 产品边界

系统聚焦"文本智能分析"本身，不负责：

- 网页爬取或数据采集
- 大规模训练平台或模型标注平台
- 证据链外部存储治理
- 完整案件管理系统

产品定位是"分析引擎 + 交互工作台"，而不是完整办案平台或通用爬虫工具。
